# 配置驱动型后台原型框架 - 产品需求文档

## 概述
- **摘要**：一个基于React 18 + TypeScript + Vite + Ant Design 5的配置驱动型后台原型框架，专为产品经理原型验证设计，通过修改JSON配置文件即可动态生成完整的列表页，无需修改前端组件代码。
- **目的**：为产品经理提供一个零代码的原型验证工具，通过配置JSON文件快速生成后台列表页，验证产品设计和用户流程。
- **目标用户**：产品经理、UI/UX设计师、前端开发人员。

## 目标
- 实现通用列表组件 `GenericListPage`，支持通过JSON配置驱动渲染
- 支持多种列类型渲染，包括文本、链接、货币、日期时间、标签和操作按钮
- 实现前端内存级分页、搜索筛选和删除模拟功能
- 支持Vite热更新，修改JSON配置后自动刷新页面
- 提供完整的示例配置和路由设置

## 非目标（范围外）
- 无需实现登录鉴权功能
- 无需连接真实API，使用前端模拟数据
- 不支持复杂的业务逻辑处理
- 不包含完整的后台管理系统功能，仅聚焦于列表页生成

## 背景与上下文
- 产品经理在原型验证阶段需要快速测试不同的列表页设计和数据结构
- 传统的前端开发流程需要修改代码才能调整页面结构，效率低下
- 配置驱动的方式可以让非技术人员也能参与原型验证

## 功能需求
- **FR-1**：通用列表组件 `GenericListPage`，接收路由传入的JSON配置路径，自动解析并渲染搜索区、数据表格、分页器和操作栏
- **FR-2**：内置列类型渲染引擎，支持 `text`、`link`、`currency`、`datetime`、`tag`、`actions` 类型
- **FR-3**：标签（tag）渲染逻辑，支持从配置中读取颜色和样式
- **FR-4**：实现前端内存级分页、搜索筛选和删除模拟功能
- **FR-5**：支持Vite热更新，修改JSON配置后自动刷新页面
- **FR-6**：提供完整的路由配置和示例配置文件

## 非功能需求
- **NFR-1**：代码质量要求，使用TypeScript确保类型安全，无类型报错
- **NFR-2**：UI设计要求，严格遵循Ant Design 5规范，确保界面美观、一致
- **NFR-3**：性能要求，页面加载和操作响应迅速，模拟网络延迟合理
- **NFR-4**：可维护性要求，代码结构清晰，注释详细，便于扩展

## 约束
- **技术**：React 18 + TypeScript + Vite + Ant Design 5 + React Router 6
- **业务**：专注于原型验证，不处理生产环境的复杂业务逻辑
- **依赖**：仅使用指定的技术栈，不引入额外的第三方库

## 假设
- 产品经理具备基本的JSON编辑能力
- 开发环境已配置好Vite和相关依赖
- 用户了解Ant Design的基本组件和交互模式

## 验收标准

### AC-1：通用列表组件渲染
- **给定**：提供有效的JSON配置文件
- **当**：访问对应的路由路径
- **则**：页面应正确渲染搜索区、数据表格、分页器和操作栏
- **验证**：`human-judgment`

### AC-2：列类型渲染
- **给定**：配置文件中定义了不同类型的列
- **当**：页面加载时
- **则**：每种类型的列应按照预期渲染，包括文本、链接、货币、日期时间、标签和操作按钮
- **验证**：`human-judgment`

### AC-3：标签渲染逻辑
- **给定**：配置文件中定义了标签的字典映射
- **当**：页面加载时
- **则**：标签应根据配置显示正确的颜色和样式
- **验证**：`human-judgment`

### AC-4：前端交互功能
- **给定**：页面已加载
- **当**：执行搜索、分页、删除操作
- **则**：操作应正确执行，并有适当的加载反馈
- **验证**：`human-judgment`

### AC-5：热更新功能
- **给定**：页面已加载
- **当**：修改并保存JSON配置文件
- **则**：页面应自动刷新，显示更新后的配置
- **验证**：`human-judgment`

### AC-6：代码质量
- **给定**：项目代码已实现
- **当**：运行TypeScript类型检查
- **则**：应无类型错误
- **验证**：`programmatic`
